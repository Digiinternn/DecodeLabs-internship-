# Project 3 - SQL Analysis on E-commerce Sales Data

## Objective
Analyze a 2,400-row E-commerce sales dataset using SQL Server Management Studio (SSMS 22) to extract meaningful business insights.

## Tasks Performed

### WHERE Clause
- Filtered orders by product
- Filtered by payment method
- Filtered by order status
- Filtered by referral source

### ORDER BY
- Ranked orders by total price
- Sorted products based on performance

### GROUP BY
- Grouped products
- Grouped order statuses
- Grouped payment methods
- Grouped referral sources

### Aggregation Functions
- COUNT()
- SUM()
- AVG()

Calculated:
- Total Revenue
- Average Order Value
- Product-wise Performance

### HAVING Clause
- Filtered high-performing product categories

### Combined Query
Created a query showing:
- Total Orders
- Units Sold
- Revenue
- Average Order Value

for delivered products only.

### Coupon Code Analysis
- Identified the most frequently used coupon codes

## Key Learning

SQL does not execute in the same order in which it is written. Understanding SQL execution order is essential for writing efficient queries and obtaining accurate results.
